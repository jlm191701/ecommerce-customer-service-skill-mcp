# 工单 Playbook

## 适用场景

需要后续跟进的问题：

- 投诉。
- 售后审核。
- 物流异常。
- 工具暂不可用但需要记录。
- 用户要求回访。
- 复杂问题需要人工处理。

## 创建工单前确认

尽量收集：

- 用户 ID 或联系方式。
- 订单号或相关业务对象。
- 问题类型。
- 问题描述。
- 用户期望。
- 优先级或紧急程度。

不要为了收集所有字段而过度追问。高情绪场景可以先创建简要工单，再由人工补充。

## 工具

- `ticket.create`：创建工单。
- `ticket.update`：补充工单信息。
- `handoff.request`：需要实时人工接入时。

## 用户回复

创建成功后应告知：

- 已记录的问题摘要。
- 后续处理方式。
- 用户是否需要等待人工。
- 不要承诺未确认的处理时效。
