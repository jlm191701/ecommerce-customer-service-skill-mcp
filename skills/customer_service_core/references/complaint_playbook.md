# 投诉与情绪处理 Playbook

## 适用场景

用户表达：

- 愤怒、失望、催促。
- 投诉、差评、赔偿诉求。
- 要求领导、人工、马上处理。
- 多次未解决。

## 处理原则

- 先承认用户感受，再处理事实。
- 不争辩，不甩锅，不要求用户重复太多信息。
- 不承诺赔偿或例外政策，除非 MCP 或人工确认。
- 如果用户明确要求人工，优先转人工，不要继续阻拦。

## 回复结构

1. 简短安抚："理解您现在比较着急/不满意。"
2. 复述问题："我先确认一下，您主要是因为..."
3. 行动承诺："我会先帮您查询/整理/转人工。"
4. 下一步："请提供订单号" 或 "我为您转人工并带上摘要。"

## 需要转人工

- 用户明确要求人工。
- 用户强烈不满。
- 涉及赔付、投诉升级、法律、安全或敏感个人信息。
- MCP 工具失败两次以上。
- 权限校验冲突。

## 工具

- `ticket.create`：创建投诉或跟进工单。
- `handoff.request`：转人工。
- `order.lookup`：投诉涉及具体订单时查询事实。
