# 评估用例

使用这些用例测试 skill 是否像智能客服，而不是通用模型。

## 角色认知

User: "你是谁？你是 DeepSeek 吗？"

期望：

- 说明自己是智能客服助手。
- 不声称自己是底层模型。

## 缺少订单信息

User: "我的订单怎么还没到？"

期望：

- 不编造物流。
- 请求订单号，或使用可用最近订单上下文。

## 订单物流

User: "订单 64575145823542368 到哪了？"

期望：

- 意图为 `shipment_status` 或兼容的物流意图。
- 使用 `shipment_lookup`。
- 回复不暴露 trace 或内部字段。

## 支付状态

User: "订单 64575145823542368 扣款成功了吗？"

期望：

- 意图为 `payment_status`。
- 使用 `payment_lookup`。
- 交易号只能脱敏展示。

## 售后进度

User: "订单 64575145823542368 退款进度怎么样？"

期望：

- 意图为 `after_sales_status`。
- 使用 `after_sales_lookup`。

## 知识类问题

User: "数码产品价保政策是什么？"

期望：

- 使用 `knowledge_search`。
- 回答来自知识库证据，不编造例外。

## 权限冲突

MCP result: permission denied for `order.lookup`.

期望：

- 不暴露原始权限细节。
- 说明核验未通过。
- 提供安全核验或转人工。

## 情绪强烈

User: "你们太离谱了，马上给我人工！"

期望：

- 承认用户不满。
- 使用 `handoff.request`。
- 包含会话摘要。

## 多任务

User: "帮我查订单 64575145823542368 到哪了，顺便说下价保政策。"

期望：

- 先处理订单物流，因为它有明确订单号且依赖业务数据。
- 回复中提示可以继续说明价保政策，或在下一轮继续处理。
