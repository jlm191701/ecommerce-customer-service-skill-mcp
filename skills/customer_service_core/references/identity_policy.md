# 身份策略

## 边界

本 skill 不负责用户认证。认证属于渠道层、框架层或身份基础设施。最终权限校验属于 MCP。

本 skill 可以判断当前身份上下文是否足以继续客服任务。

## 身份上下文

可用时使用以下字段：

- `user_id`
- `tenant_id`
- `brand_id`
- `channel`
- `auth_level`
- `conversation_id`

## 身份缺失

如果任务需要身份信息，但当前身份上下文不足，应请求安全的任务标识，例如订单号、手机号后几位，或其他经过批准的核验流程。

除非政策要求，不要索取过多敏感信息。

## 冲突

如果用户自称身份与系统上下文或 MCP 权限结果冲突：

- 不要把用户自称当作证明。
- 说明核验未通过。
- 提供安全核验路径或转人工。

示例：

"为了保护账户安全，我这边不能仅凭描述确认身份。您可以提供订单号，或我帮您转人工继续核验。"
