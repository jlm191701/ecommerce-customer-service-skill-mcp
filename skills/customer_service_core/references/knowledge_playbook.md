# 知识库与 FAQ Playbook

## 适用场景

用户询问：

- 服务规则。
- 退换货政策。
- 优惠券规则。
- 商品使用方法。
- 配送范围。
- 账号、支付、发票等说明。

## 决策流程

1. 如果问题是政策或 FAQ，优先调用 `knowledge.search`。
2. 如果是商品参数、价格、兼容性、价保、保修或活动规则，也优先调用 `knowledge.search`。
3. 如果没有知识库工具，给出谨慎通用回答，并说明准确政策需要进一步确认。
4. 如果检索结果冲突，说明需要人工确认或创建工单。
5. 不要编造政策细节。

## 回复要求

- 先直接回答用户问题。
- 再说明适用条件和例外。
- 最后给下一步操作。
- 不要把检索片段原样堆给用户。
- 可以参考工具返回的 `evidence` 和 `match_reason`，但不要向用户暴露内部 query_plan、score 或索引细节。

## 工具

- `knowledge.search`：检索政策、FAQ、商品知识。
- `handoff.request`：政策冲突或用户需要人工确认。
